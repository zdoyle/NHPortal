What is an Ad Hoc Query?  

Sometimes predefined reports don’t answer all of your business questions. Ad hoc query capability lets users easily create their own reports to gain better business insight. Gordon-Darby presents a view that allows you to explore your data in a familiar context – without requiring any knowledge of the underlying database technology or structure, and without administrative intervention.

Constructing an Ad-Hoc query

Define Fields To Include:

The first step in creating a query is to determine what fields are to be displayed in the resultant report.  This is easily accomplished by selecting the 'Include' box next to the desired field.  You may scroll through the Ad Hoc Query Builder window to see the entire record.  You may also 'jump' to particular sections of the record by clicking on the corresponding links on the left side of the window.

Enter Criteria:

After selecting the desired fields to include in the query, the user may enter the criteria they want into the 'Criteria' field.  For example, if you are interested in data for the make 'Ford', you would enter the word 'Ford' in the 'Vehicle Make' field.  The user may also include multiple criteria in a single field by entering a semi-colon ( ; ) between the criteria within the field.  For example, if you are interested in makes 'Ford' or 'Honda', you would enter 'Ford;Honda' in the criteria field for 'Vehicle Make'.

The 'Criteria' field may also include relational operators, such as; <,>,<=,>=.  For example, if you are interested in data for vehicles with greater than six cylinders, you would enter '>6' in the criteria field for 'Vehicle Number Cylinders'.  Since an exact match is the default, the '=' sign will never be used by itself.

An asterisk ( * ) may be entered for 'Wildcard' purposes.  For example, if you enter 'F*' into the criteria field for 'Vehicle Make', the resultant table will return all vehicle makes that begin with the letter F (Ferrari, Fiat, Ford, etc).

The operator ( - ) may be used for between operations. For instance, entering 'F-H' in the 'Make' field would return all vehicles with makes starting with F, G, and H. The Wildcard operator ( * ) is not needed when using the Between operator.

Date fields must be entered in the following format 'YYYYMMDD'.  For example, if you want to search from November 1, 2001 to December 20, 2001, you would enter '20011101-20011220' in the criteria section of that field.  Time fields use the following format 'HHMMSS'.  If seconds are omitted, the query will operate on hour and minute fields only.

Users may also use the Not ( ! ) operator to exclude values from a query. The '!' operator must simply be placed in front of the value(s) to exclude. Entering '!FORD' in the Make field will return all vehicles with makes that are not 'FORD'. You may also use the Not operator in conjunction with the Wildcard and Between operators. As a variation of the previous query, '!F*' could be entered in the 'Vehicle Make' field to return all vehicles whose makes do not start with 'F'. The phrase '!C-H' would be entered into the 'Vehicle Make' field to return all vehicles other than those with makes starting with C,D,E,F,G, and H.

When selecting multiple fields from a drop-down box, you must hold down the 'Control' key while clicking multiple criteria.

Choose Aggregate Values:

Users may select aggregate values for each field in the New Hampshire record.  

Count: Returns the count of items that match the value(s) for the specified field.
Sum: Returns the mathematical sum of all the items that match the value(s) for the specified field. This option only available for numeric fields.
Min: Returns the smallest number in a set of values for the specified  field.
Max: Returns the largest number in a set of values for the specified field.
Average: Displays an average taken by the sum of all the returned values divided by the count of values for that field.  This option only available for numeric fields.

 

Example Ad Hoc Query Question :
For all vehicle makes that start with 'F' or 'C' we would like to see: the names of those makes,  the names of the different models associated with each make, a count of the number of occurrences of each model;  as well as, the minimum, maximum and average model year for each Make/Model combination.

Free Form SQL Editor

Clicking on the "Free Form SQL Editor" hyperlink will take you to the Free Form SQL Editor. This tool may be used to create custom SQL queries that are not possible through the Ad Hoc Query Builder. We do not recommend using this tool unless you have a good understanding of basic SQL. There is no error checking performed on queries entered with this tool. The SQL edit window begins with the table name needed to access the VID Data Warehouse. Simply modify the rest of the Select statement however you like. A list of the available columns is available via the "View VID Column Identifiers" hyperlink at the top of the page. Enclose all criteria in apostrophes ( ' ) and column headers in double quotes ( " ). Once your query is completed, click the "Submit" button to perform your query and view the results. A small portion of the query will be displayed. If you are satisfied with the results, they may be downloaded in CSV format by clicking the Submit button labeled "Download Query Data".
